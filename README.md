# Simpler Banking – Agile Banking System
- Course: CMP9134M – Advanced Software Engineering
- Institution: University of Lincoln
- Grade Received: 73%
- Language: Python
- Framework: Django
- Methodology: Agile (Scrum)

# Overview
This project documents the Agile development of a full-stack banking web application called "Simpler Banking". The system provides user-friendly banking functionality, including account management, financial transactions, account services, and security compliance.

The report focuses on:
- Agile methodology adoption (Scrum, XP principles)
- Project planning (Gantt charts, sprint boards, user stories)
- Prototype design (UML, ER diagrams, GUI mockups, storyboards)
- Version control (Git/GitHub)
- Automated testing (Django UnitTest framework)
- Critical evaluation of software engineering tools and techniques
- Social, ethical, and entrepreneurial impact analysis

# Key Features
Account Management
- Create and set up new banking accounts (Personal/Business)
- Edit account details (user information, account type)
- Close an account

Financial Transactions
- Deposit cash with receipt generation
- Withdraw cash with real-time balance update
- Transfer funds between accounts within the same bank
- Transaction history with filters (date, type, amount)

Account Services
- View detailed account information (balance, recent transactions, account type)
- Request and manage services (chequebooks, debit/credit cards)
- Set up and manage recurring payments and direct debits

Security & Compliance
- Robust authentication mechanisms
- Data encryption for sensitive information
- Compliance with financial regulations and data protection laws

Optional Features (Implemented)
- Customer helpdesk integration
- Fraud reporting functionality
- Investment/Loan services interface (partial)
- AI-powered financial advice (stretch goal)
